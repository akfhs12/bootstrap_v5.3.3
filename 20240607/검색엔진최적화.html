검색엔진 최적화(SEO, Search Engine Optimized)

  • !DOCTYPE html : 문서 버전 의미
  • lang="ko" : 문서 내에서 주로 사용되는 언어
  • charset="UTF-8" : 문자 표현 코드, 과거에는 "EUC-KR"로도 썼음
  • meta : 문서의 제목이나 '저자', '날짜정보', '키워드' 등을 나타내고자 할 때 사용

meta 태그는 웹서버와 브라우전간의 상호교환되는 정보를 정의하는데 사용하며, 웹페이지가 담고 있는 콘텐츠가 아닌 웹페이지 자체의 정보를 명시하기 위한 목적으로 사용되는 html태그를 의미

이러한 웹페이지의 메타 정보는 주로 html의 meta 요소를 통해 마크업을 하며, HTML문서 내에서 HEAD 요소 아래 배치되므로 일반 유저가 보게 되는 웹페이지의 디자인에는 전혀 영향을 미치지 않으며 문서가 어떤 내용을 담고 있는지, 문서의 키워드는 무엇인지, 누가 만들었는지 등 문서 자체의 특성을 담고 있다.

반면 검색엔진(naver, daum, google...)과 같은 기계들이 웹페이지를 읽어야 할 때는 메타 태그의 내용들이 해당 서비스에서 어떻게 표시될 지를 결정하는 매우 핵심적인 요소가 된다

1. SEO - TITLE

검색엔진은 주기적으로 웹사이트를 방문하여 각 웹페이지가 어떤 콘텐츠를 제공하고 있는지를 파악하고 수집하여 데이터베이스에 저장하는데 점점 지능적(AI)으로 진화하고 있다. 또한 어느 정도까지는 스스로 웹페이지의 구조와 내용을 파악할 수 있다. 그럼에도 불구하고 메타 태그를 통해 웹페이지에 대한 정보를 주도적으로 명시한다면 SEO측면에서 유리하다

예) 메인페이지인 경우 - 스타벅스에 오신 것을 환영합니다.

예) 메인페이지 아닌 경우 - 스타벅스 - 오시는 길

2. SEO - META

웹사이트 제목만 예외적으로 TITLE 요소를 사용하고 그 밖에 다른 정보는 meta 요소로 표현

meta 요소 사용시 name 속성을 통해 메타 정보의 이름을 명시하고 content 속성을 통해 메타 정보의 이름과 내용을 명시

Meta 태그의 3가지 속성
1. http=equiv='항목명'
웹 브라우저가 서버에 명령을 내리는 속성으로 name 속성을 대신해 사용할 수 있으며 HTML 문서가 응답 헤더와 함께 웹 서버로부터 웹 브라우저에 전송되었을 때에만 의미
2. content="정보값"
meta 정보의 내용을 지정
3. name="정보이름"
몇 개의 meta 정보 이름을 지정할 수 있으며 지정하지 않을 시 http-equiv와 동일한 기능

meta 태그의 종류

  • meta name="keywords" content="여성의류, 웹표준" : 검색 엔진에 의해 검색되는 단어를 지정
  • meta name="description" content="신상의류, 자신의 스타일에 맞는 의상 추천" : 검색 결과에 표시되는 문자를 지정
  • meta name="robots" content="noindex, nofollow" : 검색 로봇 제어
    • All(기본값) : "index, follow"
    • None : 해당 페이지를 수집 대상으로 함
    • Follow : 해당 페이지를 포함해 링크가 걸린 곳을 수집 대상으로 함
    • Noindex : 해당 페이지를 수집대상에서 제외
    • Nofollow : 해당 페이지를 포함하여 링크가 걸린 곳을 수집 대상에서 제외
  • meta http-equiv="content-type" content="text/html; charset=UTF-8"
  • meta name="Date=" content="2023-01-19T07:45:37+09:00" : 작성날짜(제작일)
  • meta http-equiv="X-UA-Compatible" content="IE-edge" : 브라우저 호환성을 지정
  • meta http-equiv="Subject" content="웹 표준을 위한 사이트" : 홈페이지 주제
  • meta http-equiv="Title" content="웹 표준" : 홈페이지 제목
  • meta http-equiv="Other Agent" content="name" : 책임자
  • meta http-equiv="Distribution" content="name" : 배포자
  • meta http-equiv="Copyright" content="copyright@dothome" : 저작권
  • meta http-equiv="Build" content="date" : 제작일
  • meta http-equiv="Last-Modified" content="Mon, 20 Jul 2024 19:30:30" :최종 수정일
  • meta http-equiv="Refresh" content="5; url=주소" :입력한 주소로 지정 시간 후 자동 연결

SEO 태그 - Open Graph

요즘에는 SNS와 메신저를 통해 웹페이지의 링크를 공유하는 일이 잦아지면서 메타 태그가 좀 더 넓은 분야에서 활용되고 있다. 대표적 사례로 페이스북, 카카오톡, 슬랙과 같은 앱에서 웹페이지의 링크만 입력하더라도 해당 웹페이지 콘텐츠의 미리보기가 카드 형태로 나타나는 것을 들 수 있다.

일반적으로 이러한 서비스들은 Open Graph 프로토콜이라고 불리는 업계 표준을 따라 웹페이지 콘텐츠 미리보기를 지원하고 있다 .따라서 우리도 Open Graph 프로토콜에서 정의하고 잇는 메타 태그를 적절히 사용하면 콘텐츠 미리보기에 원하는 내용이 표시되도록 어느 정도 제어가 가능하다.

Open Graph 태그의 이름은 o:로 시작

meta name="og:site_name" content="사이트명" : 사이트 제목

meta name="og:title" cpmtem"페이지 제목" : 웹페이지의 제목

meta name="og:description" content="페이지 설명" : 웹페이지의 상세 설명

meta name="og:type" content="webpage" : 웹페이지의 유형

meta name="og:url" content="http://design2010.dothome.co.kr" : 웹페이지의 주소

meta name="og:image" content="http://이미지 경로" : 웹페이지의 썸네일 이미지 주소

검색 엔진이 검색할 수 있도록 제어하는 robots.txt 파일 작성법

각 검색사이트마다 서치어드바이저라는 서비스를 통해 웹사이트 등록하기

검색사이트에 웹 사이트가 검색이 되게 하기 위해서 구글, 다음, 네이트에서는 '서치어드바이저'라는 서버를 제공하는데 정상적으로 웹서버를 운영하고 사이트가 제작된 경우라면 서비스 가입을 할 수 있고, 자체 점검을 통해 '승인'이 나면 각 검색사이트에 등록시켜주는 서비스이다.
